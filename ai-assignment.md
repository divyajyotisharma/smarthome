AI Take-Home Assignment.
 
This round is designed to evaluate not only your technical skills, but also how effectively you leverage AI tools to solve problems, make decisions, and accelerate development.
 
Assignment Overview
Please find the instructions below to get started:
 
Scenario
A home may contain multiple connected appliances, such as televisions, refrigerators, air conditioners, ovens, washers, dryers, or similar devices. Appliances may come from different vendors. Vendors can differ in API style, authentication, capabilities, metric names, rate limits, and reliability. The backend should give clients a consistent way to manage appliances, collect appliance data, and produce reports.
 
What To Build
Implement a backend application that supports the following capabilities:
register and manage appliances
collect appliance metrics on configurable intervals
keep historical appliance data
generate daily reports
generate on-demand reports for custom date ranges
expose enough API behavior for the system to be reviewed end to end
You may mock external appliance vendors. You choose the API shape, data model, persistence approach, background-job approach, and report format. Keep the implementation focused enough to
run locally and review, but complete enough to demonstrate the core workflow
 
Running The Solution
Your submission should include clear local run instructions. Reviewers should be able to start the backend, exercise the main APIs, and verify that data collection and reports work.
Include tests or a simple verification path. Seed data, sample requests, or small mock payloads are fine if they make the working behavior easier to review.
 
Submission Expectations: Kindly Submit the source code and a short README covering:
how to run the service
how to run tests or verify behavior
any assumptions or non-goals
a brief note on important design choices
AI usage note, if applicable
Do not submit only a design document. The assignment will be judged primarily on working backend code.
